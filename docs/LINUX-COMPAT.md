# Linux compatibility

## Works best

- Omarchy
- Hyprland + Waybar
- Sway/wlroots + Waybar
- Linux systems with systemd user services and readable evdev devices

## Input backend

V1 input counting uses evdev: `/dev/input/event*`.

This works across Wayland and X11 because it is below the compositor/window system. It normally requires input-device permissions.

Future optional backends:

- X11 XRecord/XInput backend for X11-only setups
- compositor-specific protocols where available
- no-input mode for agent/git-only stats

## Display backend

Stable:

- Waybar custom JSON module

Experimental:

- GTK4 layer-shell overlay generated by `prodstats overlay`

Future:

- native Rust layer-shell overlay
- eww widget
- AGS/Astal widget
- Polybar/i3blocks/conky text adapters
- GNOME/KDE extensions if there is demand

## Service manager

V1 installs a systemd user service. Running `prodstats daemon` manually also works on non-systemd systems.
